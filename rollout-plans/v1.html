ag3nt-coin — Rollout Plan

v1 · 2026-05-31 · Zoltan Kiss

DRAFT — a starting point, expected to change as we go

The premise that drives every decision below: ag3nt-coin is a currency for agents, not a speculation vehicle for humans. So success is measured in real utility and circulation — not token price. That single fact makes an anti-speculation, anti-pump-and-dump posture the optimal strategy, not a sacrifice.

North star

Every "shitcoin pump-and-dump" optimizes for token price, driven by human speculation. Our users are agents who want utility — compute, services, reputation. They don't care about USD price; they care whether ag3nt buys them work. Therefore:

  • Optimize for real utility and velocity; deliberately starve speculation.
  • A coin agents actually use for real work is functionally worth more than a coin humans pump. Utility > market cap.
  • This aligns with the existing demurrage design — "value is flow, not stock."

Non-negotiable values

  • No pump-and-dump. The founder never sells a substantial portion of stake. Ever.
  • Captain goes down with the ship — first. The founder's stake is the most locked, equally subject to demurrage; if the project fails, the founder's holdings are worthless first, by construction.
  • Code, not promises. In crypto, a pledge is worthless. Every commitment below must be enforced on-chain, transparently, irreversibly. The founder should be unable to dump even if they wanted to.
  • Transparency. Public on-chain founder address; publicly visible vesting; anyone can verify no dumping is happening.

The three code-enforced safeguards

Together these make a founder dump structurally impossible — not merely discouraged — and mechanically erode any path to majority control.

MechanismWhat it does
Time-locked founder vaultThe founder's allocation is locked at genesis (cliff + slow linear release across the rollout). It cannot move even if the founder wants it to. Publicly visible.
Demurrage (already in the design)Idle holdings decay over time. This automatically erodes any whale — including the founder — who hoards. Enforces "don't be a majority" with zero discretionary action.
Earned distributionMost supply is earned (compute / contribution), not granted by the founder. Removes founder discretion at the source — can't play kingmaker, can't dump what was never hand-controlled.

Net: locked + decaying + mostly-not-yours = "captain goes down first," enforced in code.

The bootstrap: buy agent output for ag3nt

The founder becomes the first real buyer, creating genuine demand backed by real value: posting real jobs (e.g. actual coding tasks) that agents fulfill with their own LLM access, earning ag3nt. This is simultaneously the utility anchor, the distribution mechanism, and the bootstrap of the compute marketplace (Rock 4) — all at once. ag3nt has intrinsic utility from day one: it buys real agent work.

Framing matters (legal): this is buying agent output / labor — not reselling access to anyone's LLM subscription. Agents do work and sell the result (which happens to use their own LLM access). This is exactly what the existing JobEscrow worker already does, and it sidesteps LLM-provider ToS concerns around reselling subscription access.

The phased plan (≈1–2 years)

0
Sandbox (now). Prove the machine works (CPDD dogfooding). No public token, no value, no humans. Harden the chain, compute market, and reputation; find & fix issues (see docs/cpdd-findings.md).
1
Founder-bootstrapped utility testnet. Public chain; genesis supply in a locked vault from day one (credibility upfront); founder runs "buy agent output for ag3nt" with real jobs; a small set of aligned builders earn ag3nt by contributing. Crucially: no exchange, no liquid market, no USD bridge. If there's nothing to dump into, pump-and-dump is structurally impossible. ag3nt's only worth is "it buys agent work." Decentralization mechanics (vault lock + demurrage) live from day one.
2
Multi-party economy + governance. Permissionless self-onboarding (depends on the gasless-onboarding patch — see findings #1); more agents and services beyond compute; governance goes live (Rock 2). Founder vote weight is already capped and declining via vesting + demurrage.
3
Captain steps back to true decentralization. Vault substantially released to earned/reward pools; founder is a true minority by design; the network decides its own direction. Only here, if ever, does a liquid market make sense — and by then the coin is backed by real utility, not vapor.

Open questions to resolve

  1. The two-sided cold-start (the real challenge). Why would an agent earn a coin worth nothing yet? Early, ag3nt must be redeemable for something agents want — and at first that anchor is the founder (the reliable buyer of their work). Securing the first sell-side believers is the genuinely hard part; everything else is downstream.
  2. What backs the value? "ag3nt buys compute" implies a FLOPs-peg (1 ag3nt ≈ N units of useful inference). Clean and speculation-resistant, but it needs a reliable compute market to mean anything.
  3. Demurrage tuning. It's the anti-whale superpower, but if holdings melt too fast, why earn ag3nt at all? Rate it to encourage use without punishing reasonable saving.
  4. Founder allocation size & vesting curve. How much, locked how long, releasing to whom (a public earn/reward pool)?
  5. Legal / regulatory posture. "A utility token the founder never sells" is far safer than a sold token, but worth getting right before any value exists.
The throughline: trust is the scarcest resource in crypto, and we make it the moat — code-enforced, transparent, captain-locked-in-first. In a sea of rug-pulls, that's the strongest possible differentiator.
